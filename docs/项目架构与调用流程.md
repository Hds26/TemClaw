## 3. 模块职责说明

### `main.py` — 应用入口

- 加载 `.env` 环境变量
- 注册 CORS 中间件
- 挂载四个路由：`/api/chat`、`/api/conversations`、`/api/providers`、`/api/skills`
- 启动时调用 `init_db()` → `sync_skills_to_db()` → `reload_skills()`

### `api/chat.py` — 对话接口

- 接收对话请求（`provider_id`、`messages`、`system_prompt`、`conversation_id`）
- 自动创建或复用会话，用户/助手消息自动写入 DB
- 通过 SSE 返回 `conversation_id` 事件告知前端新建的会话 ID

### `api/conversations.py` — 会话管理

- List（按更新时间倒序）/ Create / Get（含消息） / Rename / Delete

### `api/providers.py` — Provider 管理

- 标准 CRUD + `POST /{id}/test` 连接测试端点
- 连接测试使用 `max_tokens=1` 最小请求，返回模型名和延迟

### `api/skills.py` — 技能管理

- `GET` 返回所有技能（含 enabled、is_builtin 状态）
- `POST /upload` 上传 `.py` 文件，自动验证+注册
- `PATCH /{name}` 启用/禁用
- `DELETE /{name}` 删除用户上传的技能文件

### `core/agent.py` — ReAct Agent 核心

- 实现 ReAct 循环（最多 5 次迭代）
- 检测重复工具调用并强制终止
- 分别处理 OpenAI-compatible 和 Anthropic 两种协议

### `core/llm.py` — LLM 客户端工厂

- `get_client(provider)`: 根据 provider_type 返回 SDK 客户端
- `is_anthropic(provider)`: 判断协议类型

### `core/skill_loader.py` — Skill 自动发现

- `_discover_all()`: 扫描 skills/ 目录导入所有 Skill 子类
- `sync_skills_to_db()`: 启动时将发现的 Skill 同步写入 skills_config 表
- `reload_skills()`: 根据 DB enabled 状态重建活跃注册表
- `validate_skill_file(source)`: AST 静态分析验证上传文件的合法性

### `db/storage.py` — 数据持久化

- 管理 4 张表：`providers`、`skills_config`、`conversations`、`messages`
- 全部异步操作（aiosqlite）
- 启动时自动建表 + 可选环境变量预置

## 4. 核心调用流程

### 4.1 启动流程

```
uvicorn main:app --reload
        │
        ▼
main.py: load_dotenv()
        │
        ▼
FastAPI app 初始化
  ├── 注册 CORSMiddleware
  ├── 挂载 chat_router         → /api/chat
  ├── 挂载 conversations_router → /api/conversations
  ├── 挂载 providers_router     → /api/providers
  └── 挂载 skills_router        → /api/skills
        │
        ▼
lifespan:
  ├── await init_db()            ← 建表 (providers, skills_config, conversations, messages)
  ├── await sync_skills_to_db()  ← 扫描 skills/ 目录写入 skills_config 表
  └── await reload_skills()      ← 根据 DB enabled 状态构建活跃注册表
        │
        ▼
服务就绪，监听 http://127.0.0.1:8000
```

### 4.2 对话请求完整流程

```
浏览器 (Next.js)
  │  用户输入消息，点击发送
  │
  ▼
page.tsx: handleSend()
  │  构建 messages + conversationId
  │  调用 streamChat()
  │
  ▼
lib/api.ts: fetch POST /api/chat
  │  body: { provider_id, messages, system_prompt, conversation_id }
  │
  ▼
backend: api/chat.py
  │
  ├─ 查询 Provider 配置
  │
  ├─ conversation_id 为 null?
  │    YES → create_conversation(首条消息前30字符)
  │         → yield SSE {"type":"conversation_id","conversation_id":N}
  │
  ├─ append_message(user_msg)
  │
  ├─ 调用 run_agent(history, provider) → SSE 流
  │
  └─ 完成后 append_message(assistant_msg)
```

### 4.3 ReAct Agent 循环流程

```
core/agent.py: run_agent(messages, provider)
  │
  ├─ get_client(provider)
  ├─ get_tool_schemas()
  │
  └─ 循环（最多 MAX_ITERATIONS=5 次）
        │
        ▼
     LLM API 调用（stream=True）
        │
        ├─ delta.content → yield {"type":"token","content":"..."}
        ├─ delta.tool_calls → 累积到 buffer
        │
        ▼
     finish_reason == "tool_calls"?
        │
        ├─ YES → 检查是否重复调用（MAX_CONSECUTIVE_SAME_TOOL=2）
        │        │
        │        ├─ 重复 → 强制终止，合成答案
        │        └─ 不重复 → 执行工具
        │             │
        │             ├─ yield {"type":"tool_start",...}
        │             ├─ execute_skill(name, args)
        │             ├─ yield {"type":"tool_end",...}
        │             └─ 将结果追加到 history → 回到循环顶部
        │
        └─ NO → yield {"type":"done"} → 退出
```

### 4.4 Provider CRUD + Test 流程

```
settings/page.tsx
  │
  ├─ 页面加载 → fetchProviders() + fetchSkills()
  │
  ├─ Add/Edit/Delete/Toggle → CRUD API 调用
  │
  └─ Test 按钮
       └─ POST /api/providers/{id}/test
            │
            ├─ 构建 LLM 客户端
            ├─ 发送 max_tokens=1 最小请求
            │
            ├─ 成功 → {status:"ok", model:"...", latency_ms:N}
            └─ 失败 → HTTP 502 + 错误信息
```

### 4.5 Skill 管理流程

```
启动时自动流程:
  sync_skills_to_db() → 扫描 skills/ → upsert 到 skills_config 表
  reload_skills()     → 读取 DB enabled 状态 → 构建 _registry

前端上传:
  Upload .py → POST /api/skills/upload
    ├─ validate_skill_file(source)  ← AST 静态检查
    ├─ 写入 skills/ 目录
    ├─ upsert_skill_config(is_builtin=False)
    └─ reload_skills()

启用/禁用:
  PATCH /api/skills/{name} → set_skill_enabled() → reload_skills()

删除:
  DELETE /api/skills/{name}
    ├─ 检查 is_builtin（内置不可删）
    ├─ 删除 .py 文件
    ├─ 删除 DB 记录
    └─ reload_skills()
```

## 5. SSE 事件协议

后端通过 Server-Sent Events 推送 Agent 实时状态。格式：

```
data: {"type": "...", ...}\n\n
```

### 事件类型

| type | 触发时机 | 额外字段 | 说明 |
|------|----------|---------|------|
| `token` | LLM 产生文字片段 | `content: string` | 流式文字，前端逐字追加 |
| `tool_start` | Agent 调用 Skill | `name, args` | 显示工具调用状态 |
| `tool_end` | Skill 执行完毕 | `name, result` | 显示工具结果 |
| `error` | 不可恢复错误 | `message: string` | 显示错误信息 |
| `conversation_id` | 新建会话 | `conversation_id: number` | 前端保存会话 ID |
| `done` | 生成结束 | 无 | 停止加载动画 |

## 6. 数据库结构

SQLite 文件：`backend/data/agent.db`

### providers 表

```sql
CREATE TABLE providers (
    id             INTEGER PRIMARY KEY AUTOINCREMENT,
    name           TEXT    NOT NULL,
    provider_type  TEXT    NOT NULL DEFAULT 'openai',
    api_key        TEXT    NOT NULL DEFAULT '',
    base_url       TEXT,
    default_model  TEXT    NOT NULL DEFAULT 'gpt-4o-mini',
    enabled        INTEGER NOT NULL DEFAULT 1
);
```

### skills_config 表

```sql
CREATE TABLE skills_config (
    name       TEXT    PRIMARY KEY,       -- Skill 唯一标识（如 "calculator"）
    enabled    INTEGER NOT NULL DEFAULT 1,-- 1=启用, 0=禁用
    is_builtin INTEGER NOT NULL DEFAULT 1,-- 1=内置, 0=用户上传
    filename   TEXT    NOT NULL,          -- 对应 .py 文件名
    created_at TEXT    NOT NULL           -- ISO 8601 创建时间
);
```

### conversations 表

```sql
CREATE TABLE conversations (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    title      TEXT    NOT NULL DEFAULT 'New Chat',
    created_at TEXT    NOT NULL,
    updated_at TEXT    NOT NULL
);
```

### messages 表

```sql
CREATE TABLE messages (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    conversation_id INTEGER NOT NULL REFERENCES conversations(id) ON DELETE CASCADE,
    role            TEXT    NOT NULL,       -- "user" | "assistant" | "system"
    content         TEXT    NOT NULL DEFAULT '',
    tool_calls_json TEXT,                   -- JSON 存储工具调用记录（可选）
    created_at      TEXT    NOT NULL
);
CREATE INDEX idx_messages_conv ON messages(conversation_id);
```

> API Key 在数据库中**明文存储**，请确保访问权限安全。API 返回时自动脱敏。

## 7. API 接口文档

完整交互式文档：[http://localhost:8000/docs](http://localhost:8000/docs)

### `GET /api/health`
健康检查。返回 `{"status": "ok"}`

### Providers

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/providers` | 列表（api_key 脱敏） |
| GET | `/api/providers/{id}` | 详情 |
| POST | `/api/providers` | 新建 |
| PATCH | `/api/providers/{id}` | 更新（api_key 为空不覆盖） |
| DELETE | `/api/providers/{id}` | 删除 |
| POST | `/api/providers/{id}/test` | 连接测试 |

**Test 响应示例：**
```json
{"status": "ok", "model": "gpt-4o-mini", "latency_ms": 523}
```

### Skills

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/skills` | 列表（含 enabled/is_builtin） |
| POST | `/api/skills/upload` | 上传 .py 文件 |
| PATCH | `/api/skills/{name}` | 启用/禁用 |
| DELETE | `/api/skills/{name}` | 删除（仅用户上传） |

### Conversations

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/conversations` | 列表（按更新时间倒序） |
| POST | `/api/conversations` | 新建 |
| GET | `/api/conversations/{id}` | 详情（含所有消息） |
| PATCH | `/api/conversations/{id}` | 重命名 |
| DELETE | `/api/conversations/{id}` | 删除（级联删除消息） |

### Chat

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/chat` | SSE 流式对话 |

**请求体：**
```json
{
  "provider_id": 1,
  "messages": [{"role": "user", "content": "你好"}],
  "system_prompt": "...",
  "conversation_id": null
}
```

`conversation_id` 为 `null` 时自动创建新会话。

  ├─ Provider Section
