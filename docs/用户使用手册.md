## 1. 项目简介

Agent Template 是一个**可插拔 AI Agent 框架**，支持接入多种主流 LLM 服务商（OpenAI、Kimi、DeepSeek、Claude 等），并通过 Skill 插件机制赋予 AI 调用外部工具的能力（如网络搜索、数学计算、日期时间查询等）。

主要特性：
- 支持 12+ 种 LLM 服务商，含国内外主流模型
- 流式输出（SSE），边生成边显示
- ReAct 风格 Agent 循环，支持多步工具调用
- Skills 热插拔 — 文件放入目录即可，也支持前端页面上传管理
- 多 Provider 管理，随时切换，支持连接测试
- 对话历史持久化，左侧边栏管理历史会话

## 3. 快速启动

### 第一步：安装后端依赖

```bash
cd agent-template/backend
pip install -r requirements.txt
```

### 第二步：配置环境变量（可选）

```bash
cp .env.example .env
```

`.env` 文件内容说明（所有项均为可选）：

```env
DEFAULT_PROVIDER_NAME=My Kimi
DEFAULT_PROVIDER_TYPE=moonshot
DEFAULT_PROVIDER_API_KEY=sk-xxxxxxxxxxxxxxxx
DEFAULT_PROVIDER_BASE_URL=https://api.moonshot.cn/v1
DEFAULT_PROVIDER_MODEL=kimi-k2.5
```

> 环境变量只在数据库中**尚无任何 Provider** 时才会自动写入。

### 第三步：启动后端

```bash
cd agent-template/backend
uvicorn main:app --reload --port 8000
```

### 第四步：安装前端依赖并启动

```bash
cd agent-template/frontend
npm install
npm run dev
```

### 第五步：打开浏览器

访问 [http://localhost:3000](http://localhost:3000)

## 4. 配置 Provider

### 进入设置

点击左侧边栏底部的 **Settings**，或主界面头部的设置入口。

### 添加 Provider

点击 **Add Provider** 按钮，填写以下信息：

| 字段 | 说明 |
|------|------|
| **Name** | 自定义名称，如 `我的 Kimi` |
| **Provider Type** | 从下拉菜单选择服务商类型 |
| **API Key** | 从对应服务商平台获取的 API Key |
| **Base URL** | 接口地址，选择内置服务商后自动填入；使用代理/中转站时需手动修改 |
| **Default Model** | 点击下方模型标签快速选择，或手动输入模型名 |

### 支持的服务商

| 服务商 | Provider Type | 官方 Base URL | 代表模型 |
|--------|--------------|---------------|---------|
| OpenAI | `openai` | `https://api.openai.com/v1` | `gpt-4o`、`gpt-4o-mini` |
| Anthropic | `anthropic` | （SDK 内置） | `claude-3-5-sonnet-20241022` |
| Moonshot (Kimi) | `moonshot` | `https://api.moonshot.cn/v1` | `kimi-k2.5`、`moonshot-v1-32k` |
| DeepSeek | `deepseek` | `https://api.deepseek.com/v1` | `deepseek-chat` |
| Qwen（阿里云） | `qwen` | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `qwen-max` |
| Zhipu AI (GLM) | `zhipu` | `https://open.bigmodel.cn/api/paas/v4` | `glm-4` |
| 01.AI (Yi) | `yi` | `https://api.lingyiwanwu.com/v1` | `yi-large` |
| Baichuan AI | `baichuan` | `https://api.baichuan-ai.com/v1` | `Baichuan4` |

### 连接测试

每个 Provider 卡片上有一个 **烧杯图标（Test）** 按钮，点击后会向对应 API 发送一条最简请求来验证连接：

- **成功**：显示绿色提示，含模型名和响应延迟
- **失败**：显示红色提示，含错误信息

> 建议添加或修改 Provider 后先点 Test 验证连接，再进行对话。

### 编辑 Provider

点击卡片右侧 **铅笔图标** 进入编辑。

> API Key 字段留空 = 保持原密钥不变，不会被覆盖。

### 启用/禁用 Provider

点击卡片右侧 **对勾/×图标** 切换启用状态。禁用的 Provider 不会出现在对话页的选择列表中。

## 5. 对话使用

### 选择 Provider

对话页顶部下拉菜单可切换当前使用的 Provider（仅显示已启用的）。

### 发送消息

- 在底部输入框输入问题
- 按 **Enter** 发送（Shift+Enter 换行）
- 点击右侧 **发送按钮** 发送

### 停止生成

生成过程中点击红色 **停止按钮** 可立即中断。

### 系统提示词

点击输入框上方的 **System Prompt** 展开，可修改 AI 的角色设定和行为规则。默认值为：

```
You are an autonomous AI agent with access to tools.
1. Actively use available tools to obtain real-time information rather than guessing or fabricating facts.
2. You may chain multiple tool calls when needed — use one tool's output to inform the next.
3. Never call the same tool with identical arguments more than once per turn.
4. After gathering information, synthesize the results and present a clear, concise answer.
5. Reply in the same language the user uses.
```

### 工具调用显示

当 AI 需要使用 Skill 时，对话框中会显示：
- 🔧 工具调用中：`tool_start` — 显示技能名和参数
- ✅ 工具结果：`tool_end` — 显示执行结果

### 错误处理

如果发生错误（如模型不存在、API Key 无效等），错误信息会直接显示在对话气泡中，不会导致页面崩溃。

## 6. 会话管理

### 侧边栏

主界面左侧为**会话列表侧边栏**，包含：

- **New Chat** 按钮：创建新对话
- **历史会话列表**：按最近更新时间排列，点击即可切换
- **Settings** 入口：底部导航至设置页

### 操作

| 操作 | 说明 |
|------|------|
| **新建** | 点击侧边栏顶部 "New Chat" 按钮 |
| **切换** | 点击列表中的任意会话，自动加载历史消息 |
| **重命名** | 鼠标悬停在会话条目上，点击铅笔图标进入编辑 |
| **删除** | 鼠标悬停在会话条目上，点击垃圾桶图标（需确认） |
| **折叠** | 点击顶部的侧边栏折叠按钮可隐藏/显示侧边栏 |

### 自动保存

- 首次发送消息时自动创建新会话，标题取自第一条消息的前 30 个字符
- 后续消息自动追加到当前会话
- 所有对话数据存储在本地 SQLite 数据库中（`backend/data/agent.db`）

## 7. Skills（技能）管理

### 查看已加载的 Skills

- **对话页**：顶部 Skills 栏显示所有**已启用**的技能标签
- **设置页**：Skills 区域以卡片形式展示所有技能，含描述、启用状态和类型标识

### 启用/禁用 Skill

在设置页的 Skills 区域，点击每个技能卡片右侧的**开关图标**切换启用状态。禁用后的 Skill 不会被 Agent 使用。

### 通过前端上传 Skill

1. 进入设置页
2. 点击 **Upload Skill** 按钮
3. 选择一个 `.py` 文件（需符合 Skill 基类规范）
4. 系统自动验证文件内容（语法检查、是否包含 Skill 子类）
5. 验证通过后自动注册并启用

> 上传的 Skill 标记为 **user-uploaded**，可以删除；内置 Skill（built-in）不可删除。

### 删除 Skill

在技能卡片上，用户上传的 Skill 会显示**垃圾桶图标**，点击即可删除文件并取消注册。

## 8. 内置 Skills 说明

### calculator（数学计算器）

让 AI 能精确计算数学表达式，避免 LLM 的计算幻觉。

支持的运算：

| 运算符/函数 | 说明 |
|------------|------|
| `+` `-` `*` `/` | 四则运算 |
| `**` | 幂运算 |
| `%` `//` | 取模、整除 |
| `sqrt(x)` | 平方根 |
| `sin(x)` `cos(x)` `tan(x)` | 三角函数 |
| `log(x)` `log10(x)` `log2(x)` | 对数 |
| `ceil(x)` `floor(x)` | 取整 |
| `abs(x)` `round(x)` | 绝对值、四舍五入 |
| `pi` `e` | 数学常数 |

示例提问：`计算 sqrt(144) + 2**10`

### web_search（网络搜索）

让 AI 能搜索 DuckDuckGo 获取最新信息（无需 API Key）。

- **search 模式**：通用网页搜索，返回标题、链接和摘要
- **news 模式**：专用新闻搜索，适合查询最新资讯
- 支持 `timelimit` 参数控制时间范围（`d`=当天，`w`=一周，`m`=一个月）

示例提问：`今天的最新 AI 新闻有哪些？`

### datetime_info（日期时间查询）

让 AI 自主获取当前日期、时间、星期和时区信息，而非依赖后端注入。

- 支持通过 `timezone_offset` 参数指定时区（默认 UTC+8 / 北京时间）
- Agent 在需要知道当前日期时会自动调用此工具

示例提问：`今天星期几？`

## 9. 添加自定义 Skill

### 方式一：通过前端上传

在设置页点击 **Upload Skill** 按钮，选择 `.py` 文件上传。

### 方式二：文件放入目录

在 `backend/skills/` 目录下新建一个 Python 文件，重启后端后自动生效。

### 文件模板

```python
# backend/skills/my_skill.py
from skills.base import Skill

class MySkill(Skill):
    name = "my_skill"
    description = "描述这个技能做什么，以及何时应该使用它。写清楚有助于 LLM 正确决策。"
    parameters = {
        "type": "object",
        "properties": {
            "input": {
                "type": "string",
                "description": "参数说明"
            }
        },
        "required": ["input"],
    }

    def execute(self, input: str) -> str:
        result = f"处理结果：{input}"
        return result
```

### 注意事项

- `name` 必须全局唯一，不能与其他 Skill 重名
- `execute()` 方法的**参数名**必须与 `parameters` 中的 `properties` 键名一致
- `execute()` 必须返回**字符串**类型
- 文件需 `from skills.base import Skill` 并继承 `Skill` 基类
- 通过前端上传时，系统会自动进行语法检查和结构验证

## 10. 常见问题排查

### 问题：`Error: network error` / 前端报网络错误

**原因**：后端服务未启动，或端口不对。

**解决**：确认后端在 8000 端口运行：
```bash
uvicorn main:app --reload --port 8000
```

---

### 问题：`404 - The model xxx does not exist`

**原因**：Provider 配置的模型名在该 API 端点上不存在。

**解决**：
1. 进入 Settings 编辑 Provider
2. 确认 Base URL 和模型名匹配
3. 使用 **Test** 按钮验证连接
4. 查询代理支持的模型：`GET {base_url}/models`

---

### 问题：`401 - Unauthorized`

**原因**：API Key 无效或为空。

**解决**：进入 Settings 编辑 Provider，重新输入正确的 API Key 后保存。

---

### 问题：编辑 Provider 后 API Key 失效

**原因**（已修复）：编辑时 API Key 留空会清空原密钥。

**解决**：编辑时在 API Key 字段重新输入密钥，或直接留空保持不变。

---

### 问题：上传 Skill 后报错

**可能原因**：
- 文件不是有效的 Python 语法
- 文件中没有继承 `Skill` 基类的类
- 类缺少 `name` 属性

**解决**：检查文件是否符合 Skill 模板规范，参考第 9 节。

---

### 问题：Agent 搜索到的新闻不是最新的

**解决**：
- 确保 Agent 先调用 `datetime_info` 获取当前日期，再带日期关键词搜索
- 系统提示词已引导 Agent 自主链式调用工具
- 可在 System Prompt 中强调"搜索前先确认当前日期"
